static void UpdateMotionMatchingState
(
    const FAnimationUpdateContext & Context,
    const TArray< TObjectPtr< const UPoseSearchDatabase >> & Databases,
    const FPoseSearchQueryTrajectory & Trajectory,
    float TrajectorySpeedMultiplier,

Remarks

Implementation of the core motion matching algorithm

Parameters

Parameter

Description

Context

Input animation update context providing access to the proxy and delta time

Databases

Input array of databases to search

Trajectory

Input motion trajectory samples for pose search queries. Expected to be in the space of the SkeletalMeshComponent. This is provided with the CharacterMovementTrajectory Component output.

BlendTime

Input time in seconds to blend out to the new pose. Uses either inertial blending, requiring an Inertialization node after this node, or the internal blend stack, if MaxActiveBlends is greater than zero.

MaxActiveBlends

Input number of max active animation segments being blended together in the blend stack. If MaxActiveBlends is zero then the blend stack is disabled.

PoseJumpThresholdTime

Input don't jump to poses of the same segment that are less than this many seconds away.

PoseReselectHistory

Input prevent re-selection of poses that have been selected previously within this much time (in seconds) in the past. This is across all animation segments that have been selected within this time range.

SearchThrottleTime

Input minimum amount of time to wait between searching for a new pose segment. It allows users to define how often the system searches, default for locomotion is searching every update, but you may only want to search once for other situations, like jump.

PlayRate

Input effective range of play rate that can be applied to the animations to account for discrepancies in estimated velocity between the movement modeland the animation.

InOutMotionMatchingState

Input/Output encapsulated motion matching algorithm and state

bForceInterrupt

Input force interrupt request (if true the continuing pose will be invalidated)

bShouldSearch

Input if false search will happen only if there's no valid continuing pose

bDebugDrawQuery

Input draw the composed query if valid

bDebugDrawCurResult

Input draw the current result if valid
